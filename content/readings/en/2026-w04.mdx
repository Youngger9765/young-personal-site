## Cline Supply Chain Attack: One Issue Title Infected 4,000 Machines

The most disturbing security incident of the week: Cline's production release was compromised via a single Issue title.

Attack chain:
1. Attacker opened an Issue on Cline's GitHub repo with Prompt Injection hidden in the title
2. Cline's AI Issue triager executed Bash commands
3. Injected instructions made Claude install a malicious npm package
4. Malicious package stuffed 11GB of junk into GitHub Actions Cache
5. Cache exceeding 10GB triggers auto-eviction — attacker's poisoned cache replaced node_modules
6. Cline's nightly release workflow shared the same cache key and loaded the poisoned cache
7. Attacker obtained NPM publish secrets and published cline@2.3.0 (later reverted)

**Lessons**: AI-powered Issue triagers must not have shell execution privileges; GitHub Actions cache keys must not be shared across workflows.

## Multi-Agent Workflow Failure Modes

GitHub Blog and Latent Space both focused on multi-agent system reliability:

- Most failures stem from **missing structure**, not model capability gaps
- Three key engineering patterns: clear agent boundaries, deterministic guardrails (tests/type checks), BDD specs as primary deliverable
